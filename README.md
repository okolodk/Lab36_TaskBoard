## Описание проекта

TaskBoard — это интерактивное веб-приложение для управления задачами (To-Do List). Приложение реализует полный цикл CRUD-операций и позволяет:

- ✅ **Просматривать** список всех задач с сортировкой (невыполненные сверху)
- ➕ **Добавлять** новые задачи с названием и описанием
- ✏️ **Редактировать** существующие задачи прямо в интерфейсе
- ✔️ **Отмечать** задачи как выполненные (с визуальным перечёркиванием)
- 🗑️ **Удалять** ненужные задачи с подтверждением
- 🔍 **Фильтровать** задачи: все / в работе / выполненные
- 📊 **Отслеживать** прогресс выполнения (счётчик задач)

## Инструкция по запуску

### 1. Запуск бэкенда

```bash
# Перейдите в папку проекта
cd TaskBoardApi

# Восстановите зависимости
dotnet restore

# Запустите сервер
dotnet run
```

### 2. Запуск фронтенда

* Откройте файл frontend/index.html в VS Code
* Нажмите правой кнопкой → "Open with Live Server"


## Основные выводы

1. **Fullstack-разработка требует комплексного подхода.**  
   При создании TaskBoard я научился связывать frontend и backend в единую систему, где каждая часть выполняет свою роль: ASP.NET Core обрабатывает запросы и работает с базой данных, а JavaScript на фронтенде отвечает за интерактивность и отображение данных.

2. **REST API — это удобный стандарт для веб-сервисов.**  
   Использование HTTP-методов (`GET`, `POST`, `PUT`, `DELETE`) делает API интуитивно понятным и предсказуемым. Каждый метод имеет чёткую семантику:
   - `GET` — для чтения данных
   - `POST` — для создания новых ресурсов
   - `PUT` — для обновления существующих
   - `DELETE` — для удаления

3. **Entity Framework Core упрощает работу с базой данных.**  
   Вместо написания SQL-запросов вручную, я использовал LINQ и асинхронные методы:
   - `ToListAsync()` — получение списка записей
   - `FindAsync()` — поиск по первичному ключу
   - `SaveChangesAsync()` — сохранение изменений  
   Миграции позволяют автоматически создавать и обновлять структуру базы данных на основе C#-классов.

4. **Асинхронное программирование критически важно для веб-приложений.**  
   Использование `async/await` как на бэкенде (в контроллере), так и на фронтенде (в функциях `fetch`) предотвращает блокировку интерфейса и позволяет серверу обрабатывать множество запросов одновременно без задержек.

5. **Безопасность данных начинается с валидации.**  
   Я реализовал защиту на нескольких уровнях:
   - Атрибуты `[Required]` и `[MaxLength]` на бэкенде контролируют целостность данных
   - Функция `escapeHtml()` на фронтенде предотвращает XSS-атаки, экранируя специальные символы перед вставкой в HTML
